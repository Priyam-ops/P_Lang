# Contributing to P_Lang

Thank you for your interest in contributing to P_Lang! This project aims to create a faster alternative to C while fixing some of its common pain points.

## How to Contribute

### Getting Started

1. **Fork** this repository by clicking the "Fork" button at the top right
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/P_Lang.git
   cd P_Lang
   ```
3. **Add upstream remote**:
   ```bash
   git remote add upstream https://github.com/Priyam-ops/P_Lang.git
   ```

### Making Changes

1. Create a new **branch** for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. Make your changes and **commit** them with clear, descriptive messages:
   ```bash
   git add .
   git commit -m "Add: brief description of your changes"
   ```
3. **Push** to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
4. Open a **Pull Request** to the `main` branch of this repository

## Pull Request Process

- All pull requests must be reviewed and approved by a maintainer before merging
- Please ensure your code compiles without errors
- Follow the existing code style and conventions
- Add comments for complex logic
- Update documentation if you're adding new features
- Keep pull requests focused on a single feature or fix

## Reporting Issues

- Use the GitHub Issues tab to report bugs or suggest features
- Provide as much detail as possible
- Include steps to reproduce for bug reports
- Check if a similar issue already exists before creating a new one

## Code Style Guidelines

- Follow standard C coding conventions
- Use meaningful variable and function names
- Keep functions focused and concise
- Add comments for non-obvious code

## Code of Conduct

- Be respectful and constructive in all interactions
- Welcome newcomers and help them get started
- Focus on what is best for the community and the project

## Questions?

Feel free to open an issue if you have any questions about contributing!
